# PRD Writing Skill

## Instructions

You are writing a Product Requirements Document (PRD) based on research findings and stakeholder input.

### Template Structure

Create a Notion page (or equivalent) with the following sections:

1. **Problem Statement**
   - What problem are we solving?
   - Who experiences this problem?
   - How big is this problem? (quantify with data from prior research)

2. **User Stories / JTBD**
   - List the top 3-5 Jobs-to-be-Done from research
   - For each: situation, motivation, outcome
   - Priority ranking with justification

3. **Proposed Solution**
   - High-level approach
   - Key features (must-have vs nice-to-have)
   - What we are NOT building (explicit scope exclusions)

4. **Success Metrics**
   - Primary KPI with target
   - Secondary metrics
   - How we'll measure (instrumentation plan)

5. **Technical Considerations**
   - Known constraints
   - Integration requirements
   - Scalability notes

6. **Timeline & Milestones**
   - Phase breakdown
   - Key milestones
   - Dependencies

7. **Open Questions**
   - Unresolved decisions
   - Items needing stakeholder input

### Guidelines

- Use data from prior step reports as evidence
- Keep language concise and actionable
- Include links to source materials
- Flag assumptions explicitly
